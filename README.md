# 🎫 HelpDesk System - מערכת ניהול פניות ושירות לקוחות

## ✍️ תודה והערכה
עם סיום הפרויקט, אני רוצה להגיד תודה רבה למורה שרה אבער. תודה על ההסברים המעולים והברורים לאורך כל השיעורים. בזכות הדרך שבה העברת לנו את החומר, הרגשתי שיש לי את הכלים והביטחון לגשת לפרויקט הזה ולבנות את המערכת בעצמי. הלמידה של React הייתה חוויה מעצימה ומלמדת מאוד, ואני ממש מעריכה את היסודות שנתת לנו שאיפשרו לי להגיע לתוצאה הזו.---

## 💻 אודות המערכת

מערכת ה-HelpDesk היא פתרון מתקדם לניהול מערך תמיכה ושירות. היא מאפשרת ניהול מלא של מחזור חיי פנייה (Ticket), החל מרגע הפתיחה על ידי הלקוח ועד לסגירתה לאחר טיפול על ידי צוות התמיכה.

**תכונות עיקריות:**
* **ממשק משתמש אינטואיטיבי:** פיתוח מבוסס React עם עיצוב מודרני ב-Material-UI.
* **ניהול פניות:** יצירת פניות עם נושאים, תיאורים ורמות דחיפות משתנות.
* **תקשורת שוטפת:** צ'אט מובנה בתוך כל פנייה המאפשר החלפת הודעות בין הלקוח לנציג.
* **לוח בקרה חכם:** דאשבורד מרכזי המציג נתונים וסטטיסטיקות בהתאם לתפקיד המשתמש.

---

## 👥 מודל הרשאות ותפקידים

המערכת בנויה על בסיס שלוש רמות הרשאה (Role-Based Access Control):

1.  **לקוח (Customer):** * פתיחת פניות חדשות במערכת.
    * מעקב אחר סטטוס הטיפול בפניות האישיות שלו.
    * הוספת תגובות ומידע נוסף בתוך הפנייה.

2.  **נציג תמיכה (Agent):**
    * צפייה בפניות שהוקצו לטיפולו.
    * עדכון סטטוס הפנייה (לדוגמה: מ"פתוח" ל"בטיפול").
    * מענה ישיר ללקוחות בתוך מערכת התגובות.

3.  **מנהל מערכת (Admin):**
    * ניהול רוחבי של כל הפניות במערכת.
    * הקצאת פניות לנציגים ספציפיים.
    * הוספת נציגים חדשים וניהול הרשאות.

## 🛠️ טכנולוגיות וכלים

### Frontend (צד לקוח)
* **React 18/19 & TypeScript:** עבודה עם Functional Components ו-Hooks.
* **Material-UI (MUI):** שימוש בספריית רכיבים מתקדמת לעיצוב רספונסיבי ונקי.
* **Context API:** ניהול מצב גלובלי (State Management) עבור מערכת ה-Auth.
* **React Router v6:** ניהול ניתובים והגנה על נתיבים (Protected Routes).
* **Axios:** תקשורת אסינכרונית מול שרת ה-API.
* **SweetAlert2:** הצגת התראות וחלוניות אישור מעוצבות.

### Backend (צד שרת)
* **Node.js & Express:** תשתית השרת.
* **SQLite:** בסיס נתונים מהיר המאחסן את הפניות, המשתמשים והתגובות.
* **JWT:** אימות משתמשים מאובטח באמצעות טוקנים.

## 🚀 הוראות התקנה והרצה

כדי להריץ את הפרויקט במחשב המקומי, יש להפעיל את שני חלקי המערכת במקביל:

### שלב 1: הפעלת ה-Backend
1. פתחו טרמינל בתיקיית השרת: `helpdesk-api-main`.
2. התקינו את הספריות: `npm install`.
3. הריצו את השרת: `npm start`.
*השרת יפעל בכתובת:* `http://localhost:4000`

### שלב 2: הפעלת ה-Frontend
1. פתחו טרמינל נוסף בתיקיית הלקוח: `helpdesk-client`.
2. התקינו את הספריות: `npm install`.
3. הריצו את האפליקציה: `npm run dev`.
*האפליקציה תפעל בכתובת:* `http://localhost:5173`

## 📂 מבנה תיקיות הקוד

* `src/context/`: מכיל את `AuthContext.tsx` האחראי על לוגיקת ההתחברות.
* `src/components/`:
    * `login.tsx` / `Register.tsx`: דפי הכניסה וההרשמה למערכת.
    * `Dashboard.tsx`: מסך הניהול הראשי הכולל את רשימת הפניות.
    * `CreateTicket.tsx`: טופס פתיחת קריאת שירות חדשה.
    * `TicketDetails.tsx`: דף הפנייה המפורט הכולל את היסטוריית התגובות.
    * `ProtectedRoute.tsx`: רכיב אבטחה המונע כניסת אורחים לדפים מוגנים.

## 🔒 אבטחה ופרטיות

* **Token Authentication:** כל בקשה לשרת עוברת דרך Header מאובטח עם JWT.
* **Role Protection:** נתיבים רגישים (כמו הוספת נציג) חסומים למשתמשים שאינם אדמין.
* **Validation:** אימות נתונים בטפסים למניעת הזנת מידע שגוי.
